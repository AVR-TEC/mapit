## tool/ls

list repo contents

### Usage

```
mapit ls <workspace name>:
  -h [ --help ]              print usage
  -w [ --workspace ] arg     the workspace to work with
  -r [ --recursive ]         the structure should be displayed recursivly
  -m [ --map ] arg           the content within the map
  -l [ --layer ] arg         the content within the layer, needs map to be 
                             specified
  -e [ --entity ] arg        the content within the entity, needs layer to be 
                             specified
  --repository-directory arg directory to store data locally
  --url arg                  remote repository url
  --compute-local            only if remote repository with option "--url" is 
                             used
```
